Tools Reference

Tools & APIs

The agent has six registered tools. Five gather live data; one provides semantic memory recall. Each tool is a plain Python callable — no decorators, only primitive parameter types.

Tools

Tool reference

🌤️
get_weather()
tools/weather.py
Open-Meteo No API key

Fetches current conditions and a three-slot forecast for a given lat/lon. Defaults to Grand Rapids, MI.

Parameters
latitudefloat, default 42.96 (Grand Rapids)
longitudefloat, default -85.67
Returns

Multi-line string with current temperature, conditions, and three forecast periods (8 am / 1 pm / 6 pm).

📰
get_news()
tools/news.py
GNews API GNEWS_API_KEY required

Fetches top general headlines and technology/cloud/AI focused stories using two separate GNews queries.

Parameters

None — queries are hardcoded (general top + technology category with cloud/AI terms).

Returns

Up to 3 general headlines + up to 2 cloud/AI highlights. Each includes title, source, and URL.

🏈
get_sports_scores()
tools/sports.py
ESPN TheSportsDB No API key

Returns structured JSON with sports data for the three tracked teams: Detroit Lions (NFL), Toronto Blue Jays (MLB), Hamilton Tiger-Cats (CFL). ESPN is primary; TheSportsDB is the CFL fallback.

Parameters
teamsoptional — defaults to the 3 tracked teams. Validated defensively at runtime.
Returns

JSON string with per-team: recent_games, upcoming_games (includes status: "in_progress" for live games), standings, and record.

Live score detection

When a game is in progress, its entry in upcoming_games has status: "in_progress" along with current scores and an inning/period/quarter detail field. The agent checks for this before deciding what to show.

🎯
get_game_plays()
tools/sports.py
ESPN No API key

Fetches detailed play-by-play data for a specific game. Called by the agent when a user asks about recent plays, current at-bat situations, or scoring plays.

Parameters
sportstr — ESPN sport slug (e.g. "baseball")
leaguestr — ESPN league slug (e.g. "mlb")
game_idstr — ESPN game ID from scores data
Returns

Last 15 at-bats/plays, scoring plays, current situation (bases, outs, count for baseball), and game state.

📅
get_calendar_events()
tools/calendar_events.py
Google Calendar v3 Service account required

Reads upcoming events from Google Calendar using a service account — no OAuth login required. Accesses private calendars shared with the service account.

Parameters
days_aheadint, default 7 — how many days to look ahead
Returns

Formatted list of upcoming events with title, date, time, and location. Returns "Nothing scheduled" if calendar is empty.

🧠
LoadMemoryTool
google.adk.tools.load_memory_tool (ADK built-in)
Supabase pgvector ADK built-in

ADK's built-in memory recall tool. Performs a semantic similarity search against past agent sessions stored in Supabase. Called automatically by the LLM when it needs past context.

How it's wired

Passed as LoadMemoryTool() in the tools list of make_agent(). The ADK runner automatically connects it to the SupabaseMemoryService configured on the Runner.

Memory write path

After every agent run, after_agent_callback calls add_session_to_memory(), which embeds the session text and stores it in Supabase pgvector.

Lessons Learned

ADK tool design lessons

Hard-won patterns from building and debugging ADK tools with Gemini and Ollama.

Problem: Dataclasses, unions with None, and nested generics (list[MyClass], Optional[str]) generate JSON Schema with $defs, $ref, and anyOf keywords. Gemini silently rejects these schemas during function calling — the tool is never invoked, producing no output and no error message.

Symptom: Tool output missing from the briefing even though smoke tests pass fine (smoke tests call the tool directly, bypassing the LLM schema check).

Fix: Use only bare primitive types in tool signatures: str, int, float, bool. Use param=None (no type annotation) for optional parameters. Validate and reconstruct objects inside the function body.

# ❌ Bad — generates $defs/$ref schemas Gemini can't handle
def get_sports_scores(teams: list[TrackedTeam] | None = None) -> str: ...

# ✅ Good — primitive default, validation inside
def get_sports_scores(teams=None) -> str:
    if not teams or not all(isinstance(t, (TrackedTeam, dict)) for t in teams):
        teams = _DEFAULT_TEAMS
    ...

Problem: Even with untyped optional parameters, the model sometimes passes strings or other unexpected types instead of leaving them at their defaults. This is especially common when the parameter name resembles something the model has an opinion about.

Fix: Always validate incoming args at function start and fall back to safe defaults. Never trust that None means the parameter wasn't passed.

def get_sports_scores(teams=None) -> str:
    # Guard: model may pass unexpected types
    if not teams or not all(isinstance(t, (TrackedTeam, dict)) for t in teams):
        teams = _DEFAULT_TEAMS
    # proceed safely...

Pattern: Tool functions should catch all exceptions and return a descriptive error string rather than raising. If a tool raises, the agent may silently skip the section or produce a confusing hallucination.

def get_news() -> str:
    try:
        articles = fetch_headlines()
        return format_articles(articles)
    except Exception as e:
        return f"[News unavailable: {e}]"

The system prompt instructs the agent: "If a tool failed, say so briefly in that section." This way, failures are visible in the briefing output rather than silent.
