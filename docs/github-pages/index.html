ADK Daily Briefing

A morning digest agent built on Google's Agent Development Kit. It gathers live weather, news, sports, and calendar data — then delivers a concise, friendly summary to Discord every day at 7 AM ET. Also available for real-time conversation.

How it works

System at a Glance

A single long-running Discord bot drives everything — scheduled daily briefings and real-time conversation, powered by an ADK agent with five tools and optional long-term memory.

Capabilities

What it covers

Five tools gather live data from external APIs. The agent synthesizes it into a concise briefing under 2 000 characters — Discord's message limit.

🌤️
Weather

Current conditions and a morning / afternoon / evening forecast for Grand Rapids, MI.

Open-Meteo No API key
📰
News

Top headlines plus cloud and AI highlights. Up to 3 general + 2 tech/cloud stories per briefing.

GNews API
🏈
Sports

Scores, records, schedules, and standings for the Detroit Lions (NFL), Toronto Blue Jays (MLB), and Hamilton Tiger-Cats (CFL). ESPN-primary with TheSportsDB fallback.

ESPN TheSportsDB No API key
📅
Calendar

Upcoming events from Google Calendar for the next 7 days, using a service account for private calendar access.

Google Calendar v3
🧠
Memory

Long-term memory via Supabase pgvector. Past sessions are embedded and stored so the agent can recall context on demand through ADK's built-in LoadMemoryTool.

Supabase pgvector LoadMemoryTool
💬
Conversation

Responds to direct Discord messages using only the tools needed for each question. Serializes per-user with asyncio.Lock — different users run concurrently.

Discord Gateway Per-user sessions
Execution modes

Briefing vs. Conversation

🕐

Scheduled Briefing

7 AM ET

Fires automatically every morning. Calls all five tools, composes the full digest, and posts chunked messages to the configured Discord channel.

  • →Isolated "scheduler" session — resets daily
  • →Shows typing indicator while generating
  • →Chunks response at ≤ 2000 chars per message
  • →Persists session to Supabase memory
💬

Conversational

On demand

Responds to user messages in real time. Uses tools only as needed, keeps answers concise. Maintains per-user session history across the bot's lifetime.

  • →Persistent per-user sessions
  • →Recalls past briefings via LoadMemoryTool
  • →Serialized with asyncio.Lock per user
  • →Live play-by-play via get_game_plays()
